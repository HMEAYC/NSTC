### 2b. MPU6500 連接

| MPU6500 腳位 | 連接 | 說明 |
|-------------|------|------|
| VCC | 3.3V | 供電 (2.375V~3.46V) |
| GND | GND | 共地 |
| SCL | GPIO7 + 4.7kΩ pull-up to 3.3V | I2C 時脈 |
| SDA | GPIO6 + 4.7kΩ pull-up to 3.3V | I2C 資料 |
| AD0 | GND | I2C 地址 = 0x68 |
| INT | (NC, 保留) | 中斷輸出 (暫不使用) |
| CLKIN | GND (或 NC) | 外部時脈 (不使用) |

**I2C pull-up 計算**：
- 4.7kΩ @ 3.3V → I²C 匯流排電容約 50pF → 上升時間約 260ns
- 符合 400kHz (Fast Mode) 規範 (< 300ns)
### 2c. 電源管理（單一外接 16500 電池）

```
                          External 16500
                          (3.7V / 800mAh)
                               │
                          [Slide Switch]
                               │
                       ┌───────┴────────┐
                       │ BAT_RAIL       │──→ ADC (GPIO0, 分壓)
                       └───────┬────────┘
                               │
                      ┌────────┴────────┐
                      │ ESP32-C3        │
                      │ Super Mini      │
                      │ (內建 LDO)      │
                      └────────┬────────┘
                               │
                          3.3V rail

電源路徑：16500 → 開關 → ESP32-C3 Super Mini VIN → 內建 LDO → 3.3V
電池獨立充電：使用外部充電器，不經由 USB-C 或板載電路
```

> **注意**：ESP32-C3 Super Mini 開發板已內建 LDO 穩壓器（通常為 AMS1117-3.3 或類似），可直接將 3.0-4.2V 電池電壓降壓至 3.3V。無需外接 ME6211 或 SS12 二極體。

**供電模式續航對照**：

| 模式 | 電池 | 容量 | WiFi 常開 | WiFi 間歇 |
|------|------|------|----------|----------|
| 標準穿戴 | 16500 | 800mAh | ~6h | ~10h |
| 長時間場域 | 16500 × 2 替換 | 1600mAh | ~12h | ~20h |
| 低功耗模式 | 16500 | 800mAh | ~8h | ~14h |

### 2d. 電池電壓監測

```
16500 + (3.0V~4.2V)
     │
     R1 = 100kΩ (1%)
     │
     ├───── GPIO0 (ADC1_CH0, ESP32-C3)
     │
     R2 = 47kΩ (1%)
     │
    GND

V_ADC = V_BAT × R2 / (R1 + R2) = V_BAT × 47/147

量測範圍：
  V_BAT = 3.0V → V_ADC = 0.96V (放電截止)
  V_BAT = 3.7V → V_ADC = 1.18V (標稱電壓)
  V_BAT = 4.2V → V_ADC = 1.34V (充飽)

ESP32-C3 ADC 參考電壓 = 3.3V → 安全範圍內

ADC 轉換公式 (ESP32-C3, 12-bit, 衰減 11dB)：
  V_ADC = adc_raw × 3.3V / 4096
  V_BAT = V_ADC × (R1 + R2) / R2 = V_ADC × 147 / 47
```

### 2e. USB 連接（僅供程式燒錄）

```
USB-C Connector
┌────┬───────────────┬──────────────┐
│Pin │ Signal        │ Connection   │
├────┼───────────────┼──────────────┤
│A1  │ GND           │ GND          │
│A4  │ VBUS          │ NC (浮接)     │
│A6  │ D+            │ GPIO19       │
│A7  │ D-            │ GPIO20       │
│A9  │ GND           │ GND          │
│A12 │ SBU1          │ NC           │
│B4  │ VBUS          │ NC (浮接)     │
│B5  │ CC2           │ 5.1kΩ → GND │
│B9  │ GND           │ GND          │
│B12 │ SBU2          │ NC           │
│Others │ NC         │ NC           │
└────┴───────────────┴──────────────┘

CC1 (A5) 各需要 5.1kΩ pulldown 到 GND
→ 告知 USB 主機為「無源裝置」

⚠️ VBUS 不連接任何電路（電池獨立充電，不經由 USB-C）
ESP32-C3 內建 USB Serial/JTAG 控制器
→ 直接連接 D+/D- 即可，不需額外 USB-UART 晶片
```

|------|------|
| 板子尺寸 | 45mm × 35mm (穿戴式腰帶安裝) |

### 3c. 天線注意事項 (ESP32-C3)

```
┌─────────────────────────────────────┐
│  ████████████████████████████████   │  ← 頂層禁佈銅區
│                                     │  (PCB 天線下方)

│                                     │
│   ◄── 天線保持懸空，無鋪銅           │

1. ESP32-C3-MINI-1 內建 PCB 天線
   → 天線區域下方**不可鋪銅** (Keep-out zone)

   → 50Ω 微帶線 (trace width ≈ 0.5mm @ 1.6mm FR-4)
   → 天線匹配 π-type 網路 (0Ω + NC + NC)
```

## 5. 編程與測試

### 5a. USB 下載

```
USB-C → ESP32-C3 原生 USB (GPIO19/20)
        ↓
無需外部 USB-UART 轉換晶片
        ↓
按住 BOOT (GPIO9→GND) → 接上 USB → 放開 BOOT
        ↓
idf.py flash monitor    ← VSCode 或命令列燒錄
```

### 5b. 初始上電測試步驟

1. **電源檢查**：接 16500 電池 → 測開關輸出 = 3.7~4.2V
2. **LDO 輸出**：測 ME6211 VOUT = 3.3V ± 2%
3. **模組啟動**：測 ESP32-C3 EN = 3.3V，串口出現 boot log
4. **I2C 掃描**：`i2cdetect` → 應看到 0x68 (MPU6500)
5. **IMU 讀取**：執行 main.c → 監控 accel/gyro 數據
6. **WiFi 連線**：設定 SSID → 看連接狀態
7. **WebSocket**：確認數據上傳到後端
8. **電池測試**：斷開/接回 16500 電池 → 確認供電正常 → 監控 ADC 電壓下降
